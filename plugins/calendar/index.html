Le module calendrier propose 2 comportements

  1. Commandes vocales (Créations, Questions, ...)
  2. Planification de message / actions

Commandes vocales

Le fichier calendar.xml propose différentes commandes vocales:
  • Sarah ajoute/créé un rappel/rendez-vous/évènement {date} {heure}
  • Sarah rappel/préviens moi dans un {heure}/{minutes}
  • Sarah qu'est ce qui est prévu aujourd'hui/demain
Indiquant au script de créer un évènement, plannifier un rappel ou vérifier les évènements à venir

Planification

Sarah interroge toutes les X minutes le calendrier afin de déterminer si un évènement arrive (en tenant compte du rappel).
  • Si le champs "where" est renseignée avec une URL une requête est effectuée
  • Sinon le titre est vocalisée
L'interface de Google Calendar devient un tableau de bord:
  • Les évènements récurents sont très pratiques
  • Le service IFTTT est capable de créer des évènements

Configuration

Editer la configuration des 2 sous modules.

URL

Renseigner (en autre) le champs url à partir de l'url privée de Google Calendar. L'URL doit être de la forme: https://www.google.com/calendar/feeds/[EMAIL]%40gmail.com/private-[NUMBER]/full?alt=json
